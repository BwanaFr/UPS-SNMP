# Device customization

You can apply some device customization to fit your needs / company.
All customizations are made in a `Customization.hpp` file.
If this file exists, the compiler will include it and use customized features. If not, the default behaviour will be used.

## Customizing logo

You can set a custom logo by defining `CUSTOM_LOGO`.
Three variables needs to be set to provide logo data:
```cpp
const int16_t custom_logo_height = 34;              //Logo height in pixels
const int16_t custom_logo_width = 90;               //Logo width in pixels
const unsigned char custom_logo [] PROGMEM = ...    //Byte array containing logo data
```
The logo byte array can be created using [image2cpp](https://javl.github.io/image2cpp/).

## Enabling and customizing auto-configuration

The device can be configured using the button and the OLED display.
To ease the configuration using this UI, the user can set an IP suffix (a digit from 0 to 255) and the device will configure all the required parameters.
This feature is implemented by defining `HAS_AUTOCONFIG` and implementing a `AutoConfig` class.
The `AutoConfig` class have only one static function `static void applyAutoconfig(uint8_t suffix)`.
This function will be called when the user validate the auto-configuration suffix setting.

Implementation of this function is left free.

## Adding OID aliases

If you want to add some aliases to SNMP OID, you can define them in the `Customization.hpp` file.
To do so, define a `OID_ALIAS` and fill a `static const char* oidAlias[][2]` array.
The first column contains the alias value, the second contains the aliased one.
This array should end with two `nullptr` values to detect the end of it.

Adding an alias will not add it in the SNMP get-next request, and will be invisible with `snmp-walk` tool.
